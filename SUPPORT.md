## Ask the community

**[GitHub Discussions](https://github.com/krwg/cultiva/discussions)** — questions, ideas, show-and-tell.

Use Discussions when you are not sure it is a bug, or you want feedback before opening an issue.

Suggested categories:

- **Q&A** — how do I…?
- **Ideas** — feature suggestions
- **Plugin authors** — manifest and sandbox API questions (also see [Author guide](docs/PLUGIN_AUTHOR_GUIDE.md))

---

## When to open an issue

Open a **[GitHub Issue](https://github.com/krwg/cultiva/issues/new/choose)** when you have:

- A **reproducible bug** in Cultiva (include OS, version, steps)
- A **concrete feature request** with context
- A **documentation gap** that blocks users

**Do not open issues for:**

- General questions → use **Discussions**
- Official plugin bugs → [cultiva-plugins issues](https://github.com/krwg/cultiva-plugins/issues/new/choose)
- Security vulnerabilities → [SECURITY.md](SECURITY.md) (private email)
- Support for unofficial / third-party plugins
